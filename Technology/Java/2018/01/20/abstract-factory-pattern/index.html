Abstract Factory Pattern

2018-01-20

1. Abstract Factory Pattern là gì?

Provide an interface for creating families of related or dependent objects without specifying their concrete classes.

Nghĩa là: Cung cấp một interface cho việc tạo ra tập hợp objects liên quan hoặc phụ thuộc nhau mà không chỉ rõ cụ thể những classes đó.

2. Ưu, nhược điểm

a. Ưu điểm

  • Dễ quản lý code hơn do ít dùng câu lệnh rẽ nhánh (if-else, switch-case) hơn khi Factory Method tăng nhiều lên.
  • Dễ dàng mở rộng số lượng Factories mà không impact đến các phần khác.

b. Nhược điểm

  • Code có thể sẽ phức tạp hơn vì cần tạo thêm cái interfaces, classes để cài đặt được pattern này

c. Khi nào áp dụng

  • Khi muốn tạo ra một loạt các objects có liên quan hoặc phụ thuộc với nhau. Ví dụ như Address và Phone Number của VietNam sẽ là 1 pair và sẽ khác với American.
  • Abstract Factory thường được cái đặt cùng Singleton, factory method và đôi khi là prototype.

3. Implementation

  • AbstractFactory: là interface hoặc abstract class, chứa các methods tạo ra các objects thuộc 2 abstract ProductA và ProductB.
  • ConcreteFactoryX: là các classes implement các methods của AbstractFactory để tạo ra các objects cụ thể.
  • AbstractProduct: là interface hoặc abstract class, các Product cụ thể sẽ implements nó.
   public abstract class Pho {
   
       protected String meat;
   
       public void cook(String note) {
           System.out.println("Cooking Pho with " + meat + " and " + note);
       }
   
       public abstract Pho delivery();
   
       public void description() {
           System.out.println("This is " + meat + " Pho");
       }
   }
   
   public class BeefPho extends Pho {
   
       public BeefPho() {
           this.meat = "Beef";
       }
   
       @Override
       public Pho delivery() {
           //do something
           return this;
       }
   }
   
  public class ChickenPho extends Pho {
   
       public ChickenPho() {
           this.meat = "Chicken";
       }
   
       @Override
       public Pho delivery() {
           //do something
           return this;
       }
   }

   public abstract class PhoFactory {
   
      // createPho bat buoc subclasses phai implement no
      // PhoFactory khong biet duoc subclasses tao instance cho class nao.
      public abstract Pho createPho(PhoType type);
   
      public void order(PhoType type){
          Pho pho = createPho(type);
          pho.description();
      }
  }
  
  public class HaNoiPhoFactory extends PhoFactory{
  
      @Override
      public Pho createPho(PhoType type) {
          Pho pho;
          if (type == PhoType.BEEF_PHO) {
              pho = new BeefPho();
          } else {
              pho = new ChickenPho();
          }
          pho.cook("well-done meat");
          return pho.delivery();
      }
  }
  
  public class NamDinhPhoFactory extends PhoFactory{
  
      @Override
      public Pho createPho(PhoType type) {
          Pho pho;
          if (type == PhoType.BEEF_PHO) {
               pho = new BeefPho();
          } else {
              pho = new ChickenPho();
          }
          pho.cook("medium-rare meat");
          return pho.delivery();
      }
  }
  
  public class Test {
  
      public static String location = "NAM_DINH";
  
      public static void main(String[] args) {
          PhoFactory phoFactory;
          if (location.equals("NAM_DINH")) {
               phoFactory = new NamDinhPhoFactory();
          } else {
              phoFactory = new HaNoiPhoFactory();
          }
          phoFactory.order(PhoType.BEEF_PHO);
      }
  }
